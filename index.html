Pather · Timezone Standardisation · 2024–25
Resolving Timezone Inconsistencies for Global Creator Events
Timezone LogicGlobal User Requirements Edge Case MappingDST Changes Cross-Border SchedulingAutomated Detection
The Problem
Digital creators hosting international events on Pather were experiencing timezone inconsistencies that caused events to display at incorrect local times for global audiences. Edge cases including Daylight Saving Time (DST) changes and cross-border scheduling were not being handled — leading to widespread scheduling errors, missed events and declining customer satisfaction scores among international users.
My Role & Approach
Gathered global user requirements through targeted stakeholder interviews and support ticket analysis to surface the root causes. Mapped all timezone edge cases including DST transitions, UTC offset variations and cross-border scheduling scenarios. Defined requirements for automated timezone detection and learner-facing display logic, ensuring events always render in the viewer's correct local time. Collaborated with Engineering to implement and validate the solution across UAT cycles.

Edge Cases Mapped & Resolved
🕰 DST Transitions
Events created before a DST change automatically recalculate and display the correct adjusted time to all viewers — no manual update needed by the creator.
🌍 Cross-Border Scheduling
Events spanning multiple countries display both the expert's timezone and the learner's local equivalent, eliminating cross-border scheduling confusion.
📡 UTC Offset Validation
All stored event times are validated against UTC offset rules on retrieval, preventing stale or incorrect offsets from reaching the user interface.
⚠️ Invalid Timezone Fallback
If an unrecognised timezone is detected, the system defaults to UTC and prompts the user to verify their timezone — ensuring no event is displayed incorrectly.
